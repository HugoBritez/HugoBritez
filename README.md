## 🧑‍💻 A bit about me

I'm a frontend-focused engineer who ended up building full systems — not because I had to, but because I wanted to understand *everything* between the user's click and the database write.

Right now I'm part of a team building software that creates real social impact. That context shapes how I think about quality: clean UI matters when real people depend on the product.

I enjoy working on state architecture, API integration patterns, and the kind of UI problems that look simple until you're actually building them.

> *"The frontend is where the product becomes real for the user. That's worth getting right."*

## 🌟 Selected Projects

### 🏢 ERP Platform
> Complex business management system built modular from day one.

Designed a multi-module frontend architecture that lets teams work independently without stepping on each other. Backed by a microservices setup with clean API boundaries. The challenge was keeping the UI coherent as the feature set grew — that's where I focused most.

`React` `TypeScript` `ASP.NET Core` `PostgreSQL` `Docker`

---

### 💬 WhatsApp Multi-tenant Messaging SaaS
> Real-time messaging platform where state consistency is critical.

Built the frontend state layer to handle live message streams across multiple tenant sessions simultaneously without UI glitches or race conditions. Implemented a real-time sync architecture that keeps everything in order even under heavy load.

`React` `Zustand` `Node.js` `WebSockets` `MongoDB`

---

### 📦 High-Performance File Service
> JWT-authenticated file distribution microservice for production systems.

Built in Go for raw performance. Handles secure, high-throughput file delivery with minimal overhead. Integrated into a larger production platform as a drop-in service. Clean API surface, fast response times.

`Go` `JWT` `Docker` `REST`
